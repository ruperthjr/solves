## Solution file anatomy

Every `solution.dart` opens with a standard header block:

```
// {problem number}. {Problem Title}
// Difficulty: Easy | Medium | Hard
// Topic: {primary tag}
// Runtime: O(?) | Space: O(?)
//
// INTUITION:
// One or two sentences — the "aha" before any code.
```

Below the header: the solution class, then a dry run tracing the key example by hand in comments, then a pattern reference if the problem connects to a previous one.

---

## Explanation file anatomy

Every `explanation.md` contains:

- **The mental model** — what image or analogy unlocks the problem.
- **Why the data structure fits** — not just "use a stack" but why a stack.
- **Step-by-step walkthrough** — one concrete example, traced manually.
- **Complexity** — time and space, justified not just stated.
- **Connection** — which previous problem shares the same skeleton.
